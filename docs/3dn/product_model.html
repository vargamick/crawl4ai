Key Normalization Benefits

  • Eliminated Redundancy: Separated repeating groups (images, documents) into dedicated entities
  • Improved Scalability: Can handle unlimited media files and documents per product
  • Flexible Media Types: Supports images (PNG, JPG), videos, and future media formats
  • Document Versioning: Tracks multiple versions of PDFs (PDS, SDS)
  • Category Management: Hierarchical category structure with many-to-many relationships
  • Data Integrity: Foreign key relationships ensure referential integrity
  • Query Optimization: Indexed fields enable efficient data retrieval

Database Schema Summary

Tables Created:

  1. products - Core product information
  2. media - All product images and videos
  3. documents - PDF documents (PDS, SDS, etc.)
  4. categories - Product categorization
  5. distributors - Retailers and wholesalers selling products
  6. industries - Industry sectors using products
  7. product_categories - Junction table for product-category relationship
  8. product_distributors - Junction table with distributor terms and status
  9. product_industries - Junction table with relevance scores and use cases

New Relationship Features:

  • Distributor Management:
    • Track multiple distributors per product
    • Store contact information and regions
    • Manage active/inactive status
    • Track contract terms and discount tiers
  • Industry Targeting:
    • Link products to multiple industry sectors
    • Score relevance for each industry (0-1 scale)
    • Document specific use cases per industry
    • Track compliance standards and certifications
    • List typical applications within each industry
